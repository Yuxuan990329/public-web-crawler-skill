# 公开网站爬虫迁移安装说明

## 1. 当前版本定位

当前项目可以作为“可迁移项目包”使用。

它不是只靠 `SKILL.md` 就能独立运行的插件，必须连同脚本、白名单、配置和 Python 环境一起迁移。

## 2. 需要一起复制的内容

至少保留：

```text
SKILL.md
requirements.txt
使用说明.md
CBNData观察说明.md
白名单网站模板_栏目优化.xlsx
scripts/
config/
```

如果要保留当前个人 DeepSeek 配置，也需要带上：

```text
config/summary_api.local.json
```

如果是给别人使用，不建议带个人 `summary_api.local.json`，让对方使用：

```text
config/summary_api.example.json
```

## 3. 新电脑初始化

进入项目目录后运行：

```powershell
python -m pip install -r requirements.txt
```

然后检查环境：

```powershell
python scripts/check_environment.py
```

## 4. 基础模式

基础模式不调用 DeepSeek，不产生 API 费用。

```powershell
python scripts/start_crawler.py
```

或：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode excerpt --resume
```

## 5. DeepSeek 增强模式

增强模式会调用 API，会产生费用。

当前个人电脑已使用本地配置文件：

```text
config/summary_api.local.json
```

如果新电脑没有该文件，可复制模板：

```text
config/summary_api.example.json
```

并改名为：

```text
config/summary_api.local.json
```

然后填入自己的 API Key。

验证增强模式：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --whitelist 白名单网站模板_栏目优化.xlsx --sites "华经情报网" --summary-mode saved --limit-per-search 1 --limit-per-site 1 --limit-total 1 --resume --request-delay 0.1
```

## 6. 失败后继续

使用 `--resume`：

```powershell
python scripts/run_pipeline.py --topic AI --mode full --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode excerpt --resume
```

批次状态记录在：

```text
pipeline_runs/
```

## 7. 注意事项

1. 只复制 `SKILL.md` 不能运行爬虫。
2. 基础模式不需要 API。
3. 增强模式需要 `config/summary_api.local.json`。
4. 不要把个人 API Key 发给其他人。
5. 公开分发时使用 `summary_api.example.json`，不要使用 `summary_api.local.json`。
