# 公开网站爬虫使用说明

## 1. 最简单启动方式

```powershell
python scripts/start_crawler.py
```

启动后按提示选择：

- 主题
- `quick` 或 `full`
- 是否启用 DeepSeek
- 是否启用 `resume`
- 选择站点
- 是否启用候选阶段 Top N AI 精筛

## 2. 基础模式是否收费

基础模式不调用 DeepSeek，不产生 API 费用。

推荐命令：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode excerpt --resume
```

```powershell
python scripts/run_pipeline.py --topic AI --mode full --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode excerpt --resume
```

## 3. DeepSeek 增强模式是否收费

增强模式会调用 DeepSeek，会产生 API 费用。

推荐先跑 quick：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode saved --resume
```

如果需要候选阶段先用 DeepSeek 精筛：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode saved --resume --ai-filter-top-n 5 --ai-filter-min-score 60
```

## 4. CSV 字段怎么看

| 字段 | 含义 |
| --- | --- |
| `summary` | 本地原文摘录，不调用 API |
| `ai_summary` | DeepSeek 摘要 |
| `ai_category` | DeepSeek 分类 |
| `ai_reason` | DeepSeek 摘要/分类理由 |
| `ai_relevance_score` | DeepSeek 候选相关性评分 |
| `ai_keep` | DeepSeek 是否建议保留候选 |
| `ai_filter_reason` | DeepSeek 候选筛选理由 |
| `content_type` | `html` 或 `public_preview` |
| `known_limit` | 已知限制，例如 PDF 受限、完整报告收费 |
| `quality_issue` | 真实质量问题 |
| `review_required` | 是否需要人工复核 |
| `source_stage` | 来源阶段，例如 `detail`、`candidate_api` |
| `candidate_reason` | 候选发现和基础打分原因 |

## 5. API Key 怎么改

运行增强模式时可以用：

```powershell
python scripts/run_pipeline.py --topic AI --mode quick --summary-mode popup --resume
```

弹窗会询问是否沿用已保存配置；选择“否”即可修改 API URL、API Key 和模型名。

本地配置文件：

```text
config/summary_api.local.json
```

该文件已被 `.gitignore` 忽略。

## 6. 失败后怎么继续

使用 `--resume`：

```powershell
python scripts/run_pipeline.py --topic AI --mode full --whitelist 白名单网站模板_栏目优化.xlsx --summary-mode excerpt --resume
```

已完成站点会自动跳过，只跑缺失或失败站点，最后重新合并和评估。

批次记录按规范化配置指纹隔离：

```text
pipeline_runs/<topic>_<mode>_<fingerprint16>/manifest.json
```

使用 `--resume` 或无人值守运行时，以 manifest 中的 `final_path` 为最终结果事实源。未使用 `--resume` 时，以命令明确打印的输出路径为准。每次执行的文件位于 `outputs/<execution_id>/` 和 `logs/<execution_id>/`；禁止按修改时间或“最新文件”回捞结果。

## 7. 当前站点策略

| 类型 | 站点 |
| --- | --- |
| 稳定全文源 | 华经情报网、智研咨询、艾瑞咨询、国家统计局 |
| 继续观察 | CBNData |
| 公开简介源 | 前瞻网 |
| 报告发现源 | 洞见研报 |
| 不自动抓 | robots 限制、登录、收费、验证码站点 |

## 8. 迁移到其他电脑

当前项目可以作为“可迁移项目包”使用，但不能只复制 `SKILL.md`。

至少需要一起复制：

```text
SKILL.md
requirements.txt
使用说明.md
白名单网站模板_栏目优化.xlsx
scripts/
config/
```

新电脑进入项目目录后运行：

```powershell
python -m pip install -r requirements.txt
python scripts/check_environment.py
```

个人使用时，可以带上当前本地 API 配置：

```text
config/summary_api.local.json
```

给别人使用时，不要带个人 API Key。让对方参考：

```text
config/summary_api.example.json
```

再自行创建 `config/summary_api.local.json`。
